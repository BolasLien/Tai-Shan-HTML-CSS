Sass

Sass 是 CSS 的預處理器 (preprocessor)，目的在撰寫 CSS 時減少反覆重工的動作，以及更有效率的樣式結構維護，Sass 必須進行編譯 (complier) 成原生的 CSS 才能使用，也就是說瀏覽器不懂 Sass，需經過編譯才能進行閱讀。

CSS 不是程式語言，沒有流程控制、函式這些功能，Sass除了可以使用崁套規則(nested rules)、繼承(extends)、混和(mixins)這些功能，還可以使用變數(variables)、函式(functions)、流程控制(Flow Control)等等，所以也可以稱為 CSS 的程式語言。

Sass 與 Scss

Sass 分為兩種，Sass 與 Scss 都稱為 Sass。不同點如下：

副檔名 年分 語法 說明
.sass 2007年，最早的處理器 縮排語法
沒有css大括號，結尾分號		 語法不能被大眾接受，無法兼容css
.scss 推出時期less之後 塊語法
寫法跟css一致，兼容css		 受到less影響，scss可以兼容css，撰寫起來親和彈性，成為普及受歡迎的預處理器。

1. 變數(Variables)

變數使用時機，當一個值被重複使用兩次以上十，就可以考慮使用

變數可以統一定義網站規範，只需修改數值，不須再CSS原始檔一一修改數值

底線(_)和連結區塊(-)的使用可以互換。定義變數名稱是$bg_black，但可以透過$bg-black提取。

2. 嵌套(Nested)

。閃套雖然很好使用，不表示所有層級都該寫入嵌套裡面，可考慮使用最少的選擇器選擇到位置。

。過度使用崁套編譯後選擇器過長，會影響到效能的執行，也會造成CSS檔案容量過大。

。以區塊嵌套有個優點，partials的概念讓結構區塊模組化，專案維護時只需針對錯誤找到模組位置修改。

3. Extend(繼承)

。繼承可以用CSS選擇器或%開始命名。

。使用CSS選擇器，會被編譯到CSS，而%佔位符號不會被編譯到CSS。

。使用@extend可以養成習慣，統一將其放在樣式的最前面。

。相同的繼承樣式在編譯後會被使用群組選擇器合併在一起。

4. 計算

。可以作加、減、乘、除、取餘數的計算

。除法比較特別，遇到除法計算，養成用括號的習慣

。Sass的計算與Calc()不同點在於預處理前已經計算好了，Calc()是在瀏覽器的階段進行計算，會消耗瀏覽器的資源。

5. Function(函式)

@function 與 @mixin 有點像，都是在做運算處理，兩者差異在於 @function 必須使用 @return 來返回結果，而且輸出的是一個值，並非是一段 css 樣式。@mixin 則可以適用 @include 產生一段樣式。

8. 資料型態

  • 1. number(數字):數字分為有單位或無單位，1,2,3,100px
  • 2. string(字串):有引號字串"foo"、"bar"，無引號字串 baz
  • 3. color(顏色):blue,#000000,rgba(0,0,0,0.5)
  • 4. boolean(布林值):true、false
  • 5. null(空值):null
  • 6. list(像陣列):用空格或逗號作為分隔、1.5rem 1rem 0 2rem 或 "Helvetica,Arial,sans-serif"
  • 7. map(物件):類似JS的物件

9. Partials(拆分在組織結構的概念)

1. 以底線來為 Partials 檔案命名，Sass會自行判斷底線開頭的 Partials 檔，有底線的 Partials 檔不會產生 CSS 檔案。

2. 載入 Partials 方式為:@import 'variables'，partials檔的底線和附檔名可以省略。

3. 許多Partials檔，例如:Mixins等等，會被用一隻主要的Sass檔來載入他們。
